# Claude Workers

A Claude Code plugin that routes tasks to the right model tier automatically вЂ” keeping costs low by matching task complexity to model capability.

## How it works

Two layers in one plugin:

```

  в”њв”Ђв”Ђ Agent tool в”Ђв”Ђв–є code-worker  (Sonnet В· read/write В· max 30 turns)
  в”њв”Ђв”Ђ Agent tool в”Ђв”Ђв–є deep-thinker (Opus   В· read/write В· max 50 turns)
  в””в”Ђв”Ђ MCP tools  в”Ђв”Ђв–є quick_task / code_task / deep_think (text-only API calls)
```

**Native agents** (primary) вЂ” Each agent runs on its own model with its own context window and tool access. Claude delegates via the built-in Agent tool. This is the full-power path.

**MCP server** (secondary) вЂ” The MCP layer exposes the same three tiers as lightweight text-in/text-out tools. Useful for simple one-shot calls that don't need a full agent session.

`CLAUDE.md` instructs the host model to evaluate each task and prefer delegation over handling it directly.

## Worker tiers

| Agent | MCP tool | Model | Max turns | Use for |
|---|---|---|---|---|
| `quick-task` | `quick_task` | Haiku | 15 | Lookups, summarization, formatting, Q&A, log analysis |
| `code-worker` | `code_task` | Sonnet | 30 | Write/fix/refactor code, tests, code review |
| `deep-thinker` | `deep_think` | Opus | 50 | Architecture, security audits, complex reasoning |

Routing rule: **start with `quick-task`** unless the task requires code changes (в†’ `code-worker`) or deep reasoning (в†’ `deep-thinker`). Never use `deep-thinker` for routine work.

## Prerequisites

- **Node.js 18+**
- **`ANTHROPIC_API_KEY`** environment variable set to a valid Anthropic API key

## Build

```bash
npm install        # installs deps and runs tsc automatically (prepare script)
```

Or build manually:

```bash
npm run build      # compiles src/ в†’ dist/
```

The MCP server reads from `dist/server.js`. Rebuild after any changes to `src/`.

## Usage

### Claude Code CLI

```bash
export ANTHROPIC_API_KEY=sk-ant-...
claude --plugin-dir /path/to/this/repo
```

Agents and MCP tools are auto-discovered on startup. The host model will delegate tasks automatically based on the routing instructions in `CLAUDE.md`. No additional setup required.

You can also invoke workers explicitly:

```
Use the quick-task agent to summarize the error logs in app.log
Use the code-worker agent to fix the failing test in src/auth.test.ts
Use the deep-thinker agent to audit the OAuth flow for security issues
```

### Claude Desktop (Code tab)

There is no `--plugin-dir` equivalent in Desktop settings. Two options:

1. **CLI bridge** вЂ” Load with `claude --plugin-dir /path/to/this/repo`, then run `/desktop` to move the session into the Desktop app with the plugin active.
2. **Marketplace install** вЂ” Publish the plugin to a marketplace (official or self-hosted), then install through the Desktop UI: **+** в†’ **Plugins** в†’ **Add plugin**.

## Extending

To add a new worker tier:

1. **Add a native agent** вЂ” Create `agents/<name>.md` with YAML frontmatter (`name`, `model`, `tools`, `maxTurns`) and a system prompt.
2. **Add an MCP tool** (optional) вЂ” Add an entry to `src/workers.ts` with `model`, `maxTokens`, `systemPrompt`, and `description`, then run `npm run build`.
3. Update the routing table in `CLAUDE.md` so the host knows when to use the new tier.
